# Project Instructions (dongtan)

이 파일은 Gemini CLI 에이전트가 이 프로젝트에서 코드를 수정하거나 배포 관련 분석을 할 때 반드시 준수해야 하는 규칙을 담고 있습니다.

## 1. 프로젝트 구조 및 경로 규칙
- **Root Directory:** 이 프로젝트의 소스 코드는 레포지토리의 최상위(root)에 위치합니다. 하위 폴더(예: `dongtan/`)를 루트로 가정하지 마십시오.
- **Pathing (The '/' Rule):**
  - Cloudflare Pages나 다른 호스팅 환경에서 절대 경로(`/`)와 상대 경로(`./`) 설정 실수를 방지해야 합니다.
  - `vue.config.js`의 `publicPath`는 기본적으로 `'/'`로 설정되어 있으나, 배포 환경의 특성에 따라 경로 관련 수정 요청이 있을 시 이 구조를 최우선으로 고려하십시오.
  - 배포 설정(예: Cloudflare Dashboard)의 **Root Directory**는 반드시 비워두거나 `/`로 설정해야 하며, 특정 폴더명을 입력하면 빌드 오류(`Cannot find cwd`)가 발생합니다.

## 2. 배포 및 빌드
- **배포 방식:** GitHub 레포지토리의 `main` 브랜치에 코드가 푸시되면 Cloudflare Pages를 통해 자동으로 빌드 및 배포가 수행됩니다.
- 프론트엔드 빌드 도구: Vue CLI (`npm run build`)
- 결과물 위치: `dist/` 폴더
- 사전 렌더링(Prerendering): `@prerenderer/webpack-plugin`을 사용하여 주요 라우트를 미리 렌더링합니다. 관련 설정을 수정할 때 렌더링 경로가 누락되지 않도록 주의하십시오.
- **API 서버 (Workers):** `api/` 폴더의 코드는 Cloudflare Workers로 배포됩니다. (마찬가지로 GitHub 연동을 통한 자동 배포 설정을 권장합니다.)

## 3. 주의사항
- 코드를 수정할 때 파일 경로를 절대 경로로 오해하여 레포지토리 상위 구조를 무시하는 실수를 하지 마십시오.
- 모든 작업 전에 레포지토리 루트의 파일 목록을 확인하여 현재 위치를 명확히 파악하십시오.

## 4. Git 및 브랜치 관리 규칙
- **브랜치 정책:** GitHub에 코드를 업데이트할 때는 반드시 `main` 브랜치에만 반영하며, 별도의 브랜치를 생성하지 마십시오. 모든 작업은 `main` 브랜치를 기준으로 수행합니다.
- **버전 보고:** GitHub에 푸시(push)한 후에는 반드시 해당 커밋의 해시(commit hash)나 버전 정보를 사용자에게 보고하십시오.

## 5. 블로그 글 작성 규칙
- **이미지 링크:** 블로그 포스트에 이미지를 포함할 때는 반드시 `.jpg`로 끝나는 이미지 링크를 사용하십시오. 다른 확장자나 동적 링크는 렌더링 오류가 발생할 가능성이 높으므로 지양해야 합니다.

## 6. 트러블슈팅 및 장애 대응
- **API 404 에러 발생 시 (Deployment Desync):**
  - GitHub에 코드를 푸시했음에도 불구하고 API 요청이 404를 반환한다면, GitHub와 Cloudflare Workers 사이의 자동 배포가 지연되거나 실패한 상태일 수 있습니다.
  - 이 경우 `api/` 폴더에서 `npx wrangler deploy` 명령어를 통해 수동으로 강제 배포를 수행하십시오.
- **API 상태 진단:**
  - 서버의 활성화 여부 및 배포된 코드의 버전을 확인하려면 `https://dongtan-api.infiniblue.workers.dev/health` 경로에 접속하십시오.
  - 응답 JSON의 `version` 및 `timestamp`를 통해 최신 코드가 반영되었는지 확인할 수 있습니다.
- **CORS 및 라우팅 규칙:**
  - 새로운 API 라우트를 추가할 때는 `app.route` 중첩 구조보다 메인 앱 인스턴스에 직접 정의하는 것을 권장합니다. (경로 매칭 오류 방지)
  - 특정 경로(예: `/batch-delete`)가 일반적인 ID 파라미터(`/:id`)에 의해 가로채이지 않도록 라우트 정의 순서에 유의하십시오.
